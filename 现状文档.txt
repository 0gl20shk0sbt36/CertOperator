# cert-operator 项目完整现状

## 项目目标

通过 TOTP + mTLS 双层认证，经 HTTPS 从 CA 服务器获取 SSH 子证书，实现零信任远程登录，并通过证书扩展控制 sudo 权限。

## 架构

```
用户/TOTP App ──(6位码)──▶ Hermes AI ──(HTTPS + mTLS)──▶ CA 服务器 ──(SSH 证书)──▶ 目标服务器
                              │                           │                        │
                           client.cert                 ca_key.pub              ca_key.pub
                           client.key                  https_cert.pem           TrustedUserCAKeys
                           ca-https-cert.pem            client.cert (verify)
```

## 已实现的功能

### 存在问题（PAM 集成）
cert-sudo-check 手动运行正确，但 PAM 集成未完全测试通过。

**PAM 配置（目标）：**
```pam
auth sufficient pam_exec.so /usr/local/bin/cert-sudo-check
auth requisite  pam_deny.so
auth sufficient pam_unix.so nullok
@include common-auth
@include common-account
session required pam_limits.so
...
```

**当前状态：**
- 在 CA 服务器（121.196.206.66，Ubuntu 24.04）上：
  - 证书 SSH 登录 ✅（文件需命名为 `密钥路径-cert.pub`）
  - 给 aibot 添加 NOPASSWD 后 sudo 成功 ✅
  - cert-sudo-check 手动运行返回 exit=0/1 正确 ✅
  - 但 `auth sufficient pam_permit.so` 对非 root 用户不生效 ❌
  - `pam_exec.so` 调用 cert-sudo-check 未验证 ❌
  - 原因猜测：Ubuntu 24.04 的 sudo 可能不读取 `/etc/pam.d/sudo`

## 已知问题

1. **PAM 集成未完全验证** — 需要在干净的 Ubuntu 22.04 或 24.04 上用 Docker 测试
2. **SSH 证书文件名规则** — 证书文件必须命名为 `私钥路径-cert.pub`，否则 ssh -i 找不到
3. **目标服务器配置步骤** — cert-sudo-check 需要手动部署到目标服务器并配 PAM
4. **deploy.ps1** — 没有给 Windows 客户端写 PowerShell 版部署脚本
